# AppDoc Parser Standard Operating Procedure (SOP)

This guide explains how to extract, parse, and utilize AppSheet schema metadata using the custom V4 Python Parser. Following this SOP ensures that the generated metadata is accurate, highly compressed (for token efficiency), and correctly identifies Virtual Columns and System attributes.

## Prerequisites

1.  Python 3.x installed.
2.  `openpyxl` installed (`pip install openpyxl`) for spreadsheet parsing.
3.  The `parse_appdoc.py` script located in the `_Utilities` folder.

## Standard Folder Structure

Every project must organize its documentation and data schemas in a dedicated `_AppDoc` folder.

```text
Projects/{ProjectName}/
├── _AppDoc/
│   ├── AppDoc.html               <-- The raw HTML export
│   ├── {SpreadsheetName}.xlsx    <-- The raw spreadsheet backups
│   └── {ProjectName}_schema.md   <-- The clean, parsed output generated by the python script
```

## Step 1: Export Application Documentation from AppSheet

1.  Create the `_AppDoc/` directory inside your project folder if it doesn't exist.
2.  Open your AppSheet application in the editor.
3.  Navigate to **Manage > Author > Documentation**.
4.  Click the link/button to view the application documentation.
5.  Once the documentation page fully loads, save the page as an HTML file into the `_AppDoc/` directory.
    *   Right-click on the page and select **Save As...**
    *   Save it as "Webpage, Complete" or "Webpage, HTML Only" (HTML Only is preferred).
    *   **Filename MUST be:** `AppDoc.html` (rename it from the default messy ID).

## Step 2: Export Data Sources (Spreadsheets)

To accurately distinguish between physical columns and Virtual Columns (VCs), the parser needs access to the physical spreadsheet structure.

1.  Go to your primary Google Sheets.
2.  Download the entire file as an `.xlsx` file.
    *   In Google Sheets: **File > Download > Microsoft Excel (.xlsx)**
    *   Save it into the `_AppDoc/` directory.
    *   Example filename: `Orbit_Database.xlsx`

## Step 3: Run the Parser

Open your terminal or command prompt and run the parser from the project's root:

```bash
python3 /Users/ommnomi/AppSheets/_Utilities/parse_appdoc.py "_AppDoc/AppDoc.html" --compact --spreadsheet "_AppDoc/Orbit_Database.xlsx"
```

### Command Flags Explained:
*   `--compact` (or `-c`): Essential! This generates the token-optimized schema suitable for AI agents. It compresses Process Tables through inheritance and formats formulas correctly.
*   `--spreadsheet` (or `-s`): Links the `.xlsx` file to cross-reference table headers. Any column found in AppSheet but *not* in the spreadsheet is definitively tagged as a Virtual Column `[VC]`.

## Step 4: Verify the Output

The parser will generate a Markdown file in the same directory as your HTML file, usually named `{AppName}_schema.md`.

Review the file for the following:
1.  **Inheritance:** Process and Output tables should collapse into single-line references (e.g., `[Inherits all 72 columns from Table: Employee]`).
2.  **Tags:** Physical columns should have appropriate flags (`PK`, `LABEL`), and Virtual Columns should be explicitly tagged with `[VC]`. System columns like `_RowNumber` should be tagged with `[SYSTEM]`.
3.  **Enums:** Enum lists should explicitly list their constraint values.
4.  **Formulas:** Long expressions (IF, SELECT, FILTER) should be nicely indented.

## Step 5: Attach to AI Sessions

Whenever you begin a new AI-assisted coding session or ask an agent to write AppSheet formulas for this application, explicitly attach or paste the contents of `{AppName}_schema.md`. This will give the AI perfect context regarding your database structure, constraints, and relationships.
